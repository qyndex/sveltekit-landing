# Contributing

Thanks for your interest in contributing! This template is part of the
[Qyngent](https://qyngent.com) open-source template collection.

## Getting Started

1. Fork this repository
2. Create a feature branch: `git checkout -b my-feature`
3. Make your changes
4. Run tests and linting (see CLAUDE.md for commands)
5. Commit with a clear message: `git commit -m "Add feature X"`
6. Push to your fork: `git push origin my-feature`
7. Open a Pull Request

## Code Style

- Follow the existing code conventions in the project
- Run the linter and formatter before committing
- Add tests for new functionality
- Keep PRs focused — one feature or fix per PR

## Reporting Issues

- Use GitHub Issues for bug reports and feature requests
- Include steps to reproduce for bugs
- Include screenshots for UI issues
